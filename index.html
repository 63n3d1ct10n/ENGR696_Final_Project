Private machine learning with On-Device training

Goal

  • This project seeks to implement machine learning models on low-power hardware devices (eg. Microcontrollers/edge devices)

  • I will optimize and train a machine learning model called Convolutional Neural Network (CNN) for image classification on Sony Spresense microcontroller

  • To achieve on-device training, the neural network architecture will be optimized for efficient resource access by Neural Network Compression

Sample Implementation

  • I will begin by implementing the on-device training on STM32F746 microcontroller board using the opensource software by MIT HAN lab

  • The cost of one board is $57

  • These have acquired already by the MIC lab

  • STM32F746NGH6 microcontroller featuring 1 Mbytes of Flash memory and 340 Kbytes of RAM

Implementation

  • The system will be deployed on the Sony Spresense Board

  • A Sony Spresense camera board will be used to capture the image in real time for on-device training

  • TensorFlow lite: will be used to load sample models for testing successful deployment

  • The MCUNet: a framework for tiny deep learning on microcontrollers.

  • TinyEngine: a memory efficient inference library

  • Cost of board is $130, already acquired

TinyEngine

  • A memory efficient and high-performance neural network library for Microcontrollers.

  • TinyEngine is a part of MCUNet, which also consists of TinyNAS.

  • MCUNet is a system-algorithm co-design framework for tiny deep learning on microcontrollers.

  • TinyEngine and TinyNAS are co-designed to fit the tight memory budgets.

CODE STRUCTURE

  • code_generator; contains a python library that is used to compile neural networks into low-level source code (C/C++).

  • TinyEngine contains a C/C++ library that implements operators and performs inference on Microcontrollers.

  • examples; contains the examples of transforming TFLite models into our TinyEngine models.

PERFORMANCE

  • System can collect training data through the camera and/or on-board sensors

  • System can use the removable flash memory (125GB) to store data bulks of data

  • System can train CNN models to optimal accuracy

  • System can update CNN model parameters based on real-time input (On-device Training)

CONCLUSION

  • On-Device training for Machine Learning models

  • Problem (overview): Current Implementation, which is training on the cloud then deployment later, is expensive and laborious

  • Goal (overview): Implementation on Edge devices (Microcontrollers)
